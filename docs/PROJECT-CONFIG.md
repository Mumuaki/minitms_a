# Конфигурация проекта MiniTMS (единый источник правды)

**При изменении документации и скриптов всегда используйте эти значения.** Не дублируйте другие пути или репозитории.

| Параметр | Значение |
|----------|----------|
| **Размещение приложения на VPS** | `/opt/minitms` |
| **Удалённый репозиторий Git** | `https://github.com/Mumuaki/minitms_a.git` |
| **Ветка по умолчанию** | `main` |
| **Сервер (IP)** | `89.167.70.67` |
| **Пользователь SSH** | `root` |

## Где это используется

- Все скрипты, подключающиеся по SSH к серверу (путь к проекту на сервере).
- Вся документация: пути к каталогу проекта на VPS, команды `cd`, пути к `.env`, systemd unit'ы, примеры.
- Синхронизация с GitHub: `origin` = minitms_a, на сервере каталог — `/opt/minitms`. Скрипт `scripts/sync-remote-server.ps1` при неудачном `git pull` (конфликты или «untracked would be overwritten») выполняет `git reset --hard origin/main`, чтобы дерево на сервере совпало с репозиторием.

## Если нужно изменить

1. Обновите этот файл.
2. Выполните поиск по проекту по старому значению и замените во всех найденных файлах (документация, `.ps1`, `.sh`, конфиги).
