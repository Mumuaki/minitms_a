# 1. ВВЕДЕНИЕ И БИЗНЕС-ТРЕБОВАНИЯ

## 1.1 Vision Document

### 1.1.1 Введение

**Trans.EU Cargo Optimizer** — это комплексная система автоматизации логистических процессов, предназначенная для автоматического поиска, анализа и подбора грузов с портала trans.eu с целью оптимизации загрузки транспортных средств и максимизации прибыльности грузоперевозок.

### 1.1.2 Позиционирование

| Аспект | Описание |
|--------|----------|
| **Для** | Транспортных компаний, диспетчеров, логистов |
| **Которые** | Занимаются международными грузоперевозками |
| **Trans.EU Cargo Optimizer** | Система автоматизации поиска и подбора грузов |
| **Которая** | Автоматически находит, анализирует и ранжирует грузы по прибыльности |
| **В отличие от** | Ручного поиска на портале trans.eu |
| **Наш продукт** | Экономит до 80% времени диспетчера и увеличивает прибыльность рейсов |

### 1.1.3 Описание заинтересованных сторон

| Роль | Представитель | Ответственность |
|------|---------------|-----------------|
| **Владелец продукта** | Директор транспортной компании | Определение бизнес-приоритетов |
| **Конечные пользователи** | Диспетчеры, логисты | Ежедневное использование системы |
| **Администраторы** | IT-специалисты | Настройка и поддержка системы |
| **Наблюдатели** | Менеджеры, аналитики | Мониторинг показателей |

### 1.1.4 Обзор продукта

#### Возможности продукта:
1. Автоматический парсинг грузов с trans.eu
2. Интеллектуальный расчёт прибыльности
3. Визуализация маршрутов на карте
4. Управление автопарком
5. Push-уведомления о выгодных грузах
6. Интеграция с GPS-трекерами
7. Email-коммуникация с заказчиками
8. Финансовое планирование и контроль
9. Интеграция с Google Sheets
10. Многоязычный интерфейс

### МОДУЛЬ 4: Алгоритм расчёта прибыльности

| ID | Требование | Приоритет |
|----|------------|-----------|
| **FR-CALC-001** | Система должна фильтровать грузы по параметрам ТС: максимальный вес (кг), длина кузова (м), ширина кузова (м), высота кузова (м) | Критический |
| **FR-CALC-002** | Система должна рассчитывать расстояние порожнего пробега (A→B) через OpenStreetMap (OSRM) | Критический |
| **FR-CALC-003** | Система должна рассчитывать ставку €/км = Цена груза / (Расстояние A→B + Расстояние B→C) | Критический |
| **FR-CALC-004** | Система должна ранжировать грузы по убыванию ставки €/км | Высокий |
| **FR-CALC-005** | Система должна учитывать минимальную приемлемую ставку из настроек пользователя | Высокий |

**Формула расчёта:**
```
Ставка (€/км) = Цена груза (€) / (Дистанция A→B + Дистанция B→C) км

Где:
- A→B: порожний пробег до точки загрузки
- B→C: гружёный пробег до точки выгрузки
```
