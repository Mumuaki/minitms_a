# Спецификация: Loads UI Feature (spec_loads.md)

## 1. Введение
Модуль `Loads` является основным рабочим пространством диспетчера. Он отвечает за отображение списка найденных грузов, их фильтрацию, расчет рентабельности "на лету" и визуализацию маршрутов на карте.

**Ключевая особенность**: Данные отображаются относительно *реального* местоположения транспортного средства (GPS), а не абстрактного города.

---

## 2. Структура Интерфейса

### 2.1. Панель Геолокации (GPS Header)
[cite_start]Отображается в верхней части списка грузов для выбранного ТС[cite: 14, 15].
* **Строка 1**: Текущий населенный пункт + Страна (результат Reverse Geocoding).
* **Строка 2**: GPS координаты (`lat`, `lon`) + Timestamp последнего обновления.
* **Элемент управления**: Кнопка **"Обновить местоположение"** (Update Location).
    * *Действие*: Вызывает Backend API для опроса GPS-трекера, обновляет координаты и перезапускает сортировку грузов.

### 2.2. Таблица Грузов (Grid)
Отображает список предложений (`OffersNormalized`).

#### Колонки таблицы:
**Рентабельность** (Цветной индикатор).
**Маршрут**:
**Место загрузки** (Флаг страны + Город + Почтовый индекс).
**Место выгрузки** (Флаг страны + Город + Почтовый индекс).
**Расстояние**: `(A→B) + (B→C)`.
**Груз**:
**Описание** (напр., "24т, Тент").
**Даты** (Загрузка / Выгрузка).
**Финансы**:
**Цена** (EUR).
**Ставка €/км** (Расчетная).
**Действия** (Actions):
    * Открыть на карте.
    * Отправить Email.
    * Скрыть (Blacklist).
    * Принять (Создать заказ).

## 3. Логика Визуализации

### 3.1. Цветовая кодировка (Profitability Highlighting)
Каждая строка подсвечивается цветом в зависимости от расчетной ставки (€/км).

| Индикатор | Цвет (HEX) | Диапазон €/км | Значение |
|---|---|---|---|
| 🔴 **Красный** | `#FF4444` | **< 0.54** | Убыточный рейс |
| ⚪ **Серый** | `#9E9E9E` | **0.54 - 0.59** | Низкая рентабельность |
| 🟡 **Желтый** | `#FFEB3B` | **0.60 - 0.79** | Средняя рентабельность |
| 🟢 **Зеленый** | `#4CAF50` | **≥ 0.80** | Высокая рентабельность |

### 3.2. Расчетные показатели (Client-side Calc)
Для мгновенного отклика интерфейс должен уметь пересчитывать показатели при изменении настроек (без запроса к серверу, если данные уже загружены):
**Дистанция порожнего пробега (A→B)**: Расстояние от *Текущей GPS точки* до *Места загрузки*.
**Полная дистанция**: `(A→B) + (B→C)`.
**Ставка (€/км)**: `Цена предложения / Полная дистанция`.
**Операционная себестоимость**: `(Полная дистанция * CostPerKm) + ExtraCosts`.

### 3.3. Карта Маршрута (Map View)
При клике на груз или иконку карты открывается модальное окно с Google Maps.

**Маркеры**:
**A (Красный)**: Текущая GPS позиция ТС.
**B (Зеленый)**: Место загрузки.
**C (Синий)**: Место выгрузки.

**Линии**:
**Пунктирная**: Маршрут подсыла (`A → B`).
**Сплошная (Синяя)**: Маршрут перевозки (`B → C`).

## 4. Фильтры и Сортировка

### 4.1. Автоматическая фильтрация
Система скрывает грузы, которые:
* Не соответствуют габаритам кузова (`L x W x H`) выбранного ТС.
* Превышают грузоподъемность.
* Находятся в "Черном списке" (ранее скрытые пользователем).

### 4.2. Пользовательские фильтры
**Радиус поиска**: Ползунок/Input (км) от точки GPS.
**Страны**: Dropdown (только загрузка, только выгрузка).
**Исключить**: Чекбокс "Скрыть предложения без цены".

### 4.3. Сортировка
По умолчанию список отсортирован по **Убыванию ставки (€/км)**, чтобы самые выгодные предложения были сверху.
