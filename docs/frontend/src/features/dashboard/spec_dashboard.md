## 1. Описание
Главный экран системы, предоставляющий сводную информацию о текущем состоянии автопарка, финансовых показателях и уведомлениях. Агрегирует данные из модулей Finance, Fleet и Notifications.

## 2. Компоненты UI

### 2.1 KPI Cards (Финансовая сводка)
Отображение 4-х ключевых карточек "План / Факт":
1.  **Выручка (Revenue):** План (€) / Факт (€) / % выполнения.
2.  **Маржа (Margin):** План (€) / Факт (€) / % выполнения.
3.  **Пробег (Mileage):** План (км) / Факт (км) / % выполнения.
4.  **Средняя ставка (Avg Rate):** Факт (€/км).

**Визуальная индикация отклонений (Color Coding):**
* 🟢 **Зеленый:** В плане.
* 🟡 **Желтый:** Отклонение до 10% (в худшую сторону).
* 🔴 **Красный:** Отклонение > 10%.

### 2.2 Notifications Widget
Виджет последних уведомлений.
* Группировка уведомлений во избежание спама (не чаще 1 раза в 5 минут).
* Типы: "Выгодный груз", "Критическое отставание от плана", "Лимит Email".

### 2.3 Fleet Status Summary
Краткая сводка по статусам ТС (Свободен / В рейсе / На ТО / Недоступен). Возможно отображение мини-карты с кластерами машин.

## 3. Логика и State Management
Данные загружаются параллельно при монтировании компонента Dashboard.

**Вычисляемые значения:**
* *% Выполнения:* `(Факт / План) * 100`.
* *Средняя ставка:* `Фактическая выручка / Фактический пробег`.

## 4. Интеграция с API
| Метод | Эндпоинт | Описание |
| :--- | :--- | :--- |
| GET | `/reports/dashboard-stats` | Получение агрегированных данных (План/Факт) за текущий месяц. |
| GET | `/notifications/recent` | Получение списка последних непрочитанных уведомлений. |
| GET | `/fleet/status-summary` | Получение количества машин по статусам. |

## 5. Доступ
* **Administrator, Director:** Полный доступ ко всем виджетам.
* **Dispatcher:** Видит статусы ТС и уведомления, финансовые показатели могут быть скрыты или ограничены (только просмотр `👁️`).
* **Observer:** Только просмотр.
