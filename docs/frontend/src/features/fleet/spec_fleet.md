## 1. Обзор
Модуль управления автопарком (`features/fleet`) предоставляет интерфейс для управления транспортными средствами, отслеживания их текущего местоположения по GPS и установки финансовых планов. Он служит источником достоверных данных о параметрах транспортных средств, используемых в алгоритмах "Расчета рентабельности".

## 2. Функциональные требования

### 2.1 Список и Управление ТС
* **Отображение:** Табличный вид всех транспортных средств.
* **Колонки:** Регистрационный номер, Тип, Габариты (ДxШxВ), Грузоподъемность, Статус, Текущее местоположение.
* **Действия:** Добавить ТС, Редактировать ТС, Удалить ТС, Просмотр истории.
* **Валидация данных:**
    * Габариты должны быть в метрах.
    * Вес должен быть в килограммах.
    * Обязательные поля: Госномер, Тип кузова.

### 2.2 Детали ТС и Конфигурация
При добавлении или редактировании ТС (`VehicleModal`) требуются следующие данные для поддержки движка расчетов:
* **ID/Госномер ТС:** Уникальный идентификатор.
* **Тип кузова:** Выбор из списка (Тент, Бокс, Рефрижератор и т.д.).
* **Габариты:**
    * Длина (м)
    * Ширина (м)
    * Высота (м)
* **Грузоподъемность:** Макс. вес (кг).
* **Водитель по умолчанию:** (Опционально) Связь с пользователем.

### 2.3 GPS и Отслеживание местоположения
* **Интеграция:** UI должен отображать "Реальное" местоположение на основе данных GPS.
* **Ручное обновление:** Интерфейс должен содержать кнопку "Обновить местоположение".
    * *Действие:* Инициирует запрос к бэкенду для получения свежих данных от GPS-провайдера.
    * *Визуальная индикация:* Обновляет координаты и метку времени "Последнее обновление" сразу после ответа.
* **Визуализация на карте:**
    * Отображение текущей позиции ТС на компоненте карты.
    * Показ текущего адреса (Результат обратного геокодирования: Город + Страна).

### 2.4 Финансовое планирование (На ТС)
* **Ввод данных:** Возможность установки ежемесячных планов для конкретного ТС.
    * Плановая выручка (€)
    * Плановый пробег (км)
    * Плановая маржа (€)

## 3. Структура Компонентов UI

### `FleetTable.tsx`
* Использует компонент `SharedTable`.
* Рендерит бейджи статусов (Зеленый: Свободен, Красный: На ТО, Синий: В рейсе).

### `VehicleModal.tsx`
* Форма для операций Добавления/Редактирования.
* Логика валидации для габаритов (проверка числовых значений).

### `LocationWidget.tsx`
* **Отображение:**
    * Строка 1: Город, Страна.
    * Строка 2: GPS Координаты (Широта, Долгота).
* **Управление:** Кнопка "Обновить местоположение".

## 4. Управление Состоянием (Store)
* **State:** `vehicles: Vehicle[]`, `loading: boolean`, `error: string`.
* **Actions:** `fetchVehicles()`, `addVehicle(data)`, `updateVehicle(id, data)`, `syncGpsLocation(vehicleId)`.

## 5. Используемые API Endpoints
* `GET /api/v1/fleet`
* `POST /api/v1/fleet`
* `PUT /api/v1/fleet/{id}`
* `POST /api/v1/fleet/{id}/gps/sync`
