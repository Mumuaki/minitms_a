# Спецификация: Отчеты и Дашборд

**Модуль:** Frontend / Features / Reports
**Родительский документ:** [Architecture - Module 8 & 12]
**Версия:** 1.0

## 1. Обзор
Модуль отчетов агрегирует данные из источников Заказов, Автопарка и GPS для предоставления анализа "План / Факт". Он позволяет Директору и Администратору отслеживать финансовое состояние и операционную эффективность в режиме реального времени.

## 2. Функциональные требования

### 2.1 Виджеты Дашборда (План / Факт)
Дашборд должен отображать 4 ключевые карточки за выбранный период:
1.  **Выручка:** План vs. Факт (€).
2.  **Маржа:** План vs. Факт (€).
3.  **Пробег:** План vs. Факт (км).
    * *Источник:* Фактический пробег рассчитывается как разница показаний одометра GPS (Конец дня - Начало дня).
4.  **Средняя ставка:** Факт (€/км).
    * *Формула:* Фактическая выручка / Фактический пробег.

### 2.2 Визуальные индикаторы
* **Зеленый:** В плане или превышает план.
* **Желтый:** Отклонение < 10%.
* **Красный:** Отклонение > 10% (Критический алерт).

### 2.3 Детальные отчеты
* **Отчет по рентабельности:** Таблица, показывающая все завершенные заказы с рассчитанными €/км и маржой.
* **Использование автопарка:** Разбивка пробега (Груженый / Пустой) по каждому транспортному средству.

### 2.4 Функционал Экспорта
* **Действия:** Кнопка для экспорта текущего вида/данных в Excel и PDF.
* **Синхронизация с Google Sheets:** Индикатор статуса, показывающий, синхронизированы ли данные отчета с мастер-таблицей Google Sheet.

## 3. Структура Компонентов UI

### `DashboardStats.tsx`
* Сетка (Grid layout), содержащая компоненты `StatCard`.
* Принимает фильтр `period` (Текущий месяц, Прошлый месяц).

### `PlanFactChart.tsx`
* Визуализация в виде столбчатой диаграммы, сравнивающая План (Серый) и Факт (Цветовое кодирование по статусу).

### `ReportFilters.tsx`
* Выбор диапазона дат.
* Селектор транспортного средства (Все vs Конкретное ТС).

## 4. Логика и Расчеты
* **Расчет пробега:** Должен запрашивать у бэкенда агрегированную дистанцию GPS, а не просто сумму дистанций заказов.
* **Операционные расходы:**
    * *Вводные:* Стоимость €/км (фиксированная настройка) + Дополнительные расходы.
    * *Формула:* `(Общая дистанция * Стоимость за км) + Доп. расходы`.

## 5. Используемые API Endpoints
* `GET /api/v1/reports/dashboard-stats?period={date}`
* `GET /api/v1/reports/financial?from={date}&to={date}`
* `GET /api/v1/reports/export?format={pdf|xlsx}`
