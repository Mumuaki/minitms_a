# Specification: Planning Use Cases

Этот модуль отвечает за финансовое планирование, установку KPI для автопарка и отслеживание их выполнения (План/Факт).
Основан на функциональных требованиях **Module 12: Финансовое планирование**.

## 1. Set Monthly Plan (UC-004)

### Description
Позволяет Директору (Director) установить плановые финансовые показатели для конкретного транспортного средства на заданный месяц.

### Actors
* **Primary:** Director

### Input Data (`PlanDto`)
* `vehicle_id`: ID транспортного средства
* `month`: Месяц планирования (YYYY-MM)
* `target_revenue`: Плановая выручка (€)
* `target_margin`: Плановая маржа (€)
* `target_mileage`: Плановый пробег (км)

### Flow
1.  Система проверяет права пользователя (Role = Director).
2.  Система валидирует входные данные (значения должны быть > 0).
3.  Система проверяет, существует ли план для данного ТС на этот месяц.
    * Если существует: обновляет текущую запись и логирует изменение (UC-004.5).
    * Если нет: создает новую запись.
4.  Сохраняет данные через `PlanRepository`.
5.  Триггерит пересчет дэшборда.

### Business Rules
* **BR-PLAN-01**: Планы могут задаваться на будущие и текущие месяцы. Корректировка прошлых месяцев требует подтверждения администратора.

## 2. Calculate Plan vs Fact (UC-008)

### Description
Автоматический процесс агрегации фактических данных и сравнения их с плановыми показателями. Используется для отображения карточек на Dashboard.

### Flow & Calculation Logic

#### 1. Сбор фактических данных (Fact Gathering)
* **Выручка (Fact Revenue):**
    * Запрашивает `OrderRepository` для получения суммы поля `invoice_amount` по всем заказам со статусом "Completed" за отчетный месяц для данного ТС.
* **Пробег (Fact Mileage):**
    * Использует `GpsServicePort.getDailyMileage` для каждого дня месяца.
    * **Logic:** `Sum(GPS_Odometer_23:59 - GPS_Odometer_00:00)` для всех дней месяца.
* **Маржа (Fact Margin):**
    * `Fact Revenue` - `Total Expenses` (Расходы берутся из модуля Finance, если реализован, или упрощенно как % от выручки/пробега).

#### 2. Расчет производных метрик
* **Средняя ставка (€/км):**
    * Formula: `Fact Revenue / Fact Mileage`.

#### 3. Сравнение и Индикация (Status Determination)
Для каждого показателя (Выручка, Маржа, Пробег) вычисляется процент отклонения:
`Delta % = (Fact / Plan) * 100`

Система присваивает статус согласно бизнес-правилам:
* **BR-019 (Зеленый):** Отклонение < 10% (или выполнение > 90% для выручки).
* **BR-019 (Желтый):** Отклонение 10-20% (или выполнение 80-90%).
* **BR-020 (Красный):** Отклонение > 10% в негативную сторону (или выполнение < 80%).
* **BR-021 (Critical Alert):** Если отставание по выручке > 20%, формируется событие `CriticalPlanLagEvent` для уведомления Директора.

### Output (`PlanFactReportDto`)
Возвращает структуру данных для отображения 4 карточек Dashboard:
1.  💰 Выручка (План / Факт / %)
2.  📊 Маржа (План / Факт / %)
3.  🚗 Пробег (План / Факт / %)
4.  📈 Средняя ставка (Факт €/км)

## 3. Dependencies
* `OrderRepository`: Источник данных о выручке.
* `GpsServicePort`: Источник данных о фактическом пробеге.
* `NotificationPort`: Для отправки алертов при критическом отставании.
