# Use Case Specification: Cargo & Scraping

## 1. Введение
Модуль отвечает за получение данных о грузах, фильтрацию по параметрам ТС и расчет прибыльности.

## 2. Use Cases

### UC-CARGO-01: Scrape Trans.eu (Парсинг грузов)
* **Actor:** System (Scheduler) или Dispatcher (Manual).
* **Pre-conditions:** Наличие активной сессии Trans.eu, наличие координат ТС.
* **Flow:**
    1. Получить текущие координаты ТС (из модуля Fleet).
    2. Определить локацию поиска:
       * Геокодирование координат в "Страна, Индекс, Город".
       * Если точный город не найден, использовать ближайший крупный узел.
    3. Эмуляция действий пользователя:
       * Открыть `https://platform.trans.eu/exchange/offers`.
       * Установить `Loading Place`: Страна + Город.
       * **Важно:** Установить радиус поиска `+75 km` (или 5-200 км по настройке).
       * Установить даты: `Loading Date` = Текущая дата, `Unloading Date` = +1 день (или настройки пользователя).
       * Установить фильтры ТС: Вес (0.9т), Тип кузова.
    4. Сбор карточек (DOM parsing):
       * Извлечение: Откуда, Куда, Вес, Цена, Описание, ID груза.
    5. Нормализация данных и сохранение в сущность `OffersNormalized`.

### UC-CARGO-02: Calculate Profitability (Расчет прибыльности)
* **Actor:** System.
* **Trigger:** Появление новых записей в `OffersNormalized`.
* **Flow:**
    1. Для каждого груза получить координаты точки загрузки (Loading Point B) и выгрузки (Unloading Point C).
    2. Рассчитать дистанцию порожнего пробега: Точка А (ТС) -> Точка B (OSM API).
    3. Рассчитать дистанцию груза: Точка B -> Точка C.
    4. Рассчитать ставку: `€/km = Price / (Dist(A-B) + Dist(B-C))`.
    5. Присвоить цветовой тег:
       * 🔴 < 0.54
       * ⚪ 0.54 - 0.59
       * 🟡 0.60 - 0.79
       * 🟢 ≥ 0.80.

### UC-CARGO-03: Filter & List (Просмотр списка)
* **Actor:** Dispatcher.
* **Features:**
    * Сортировка по `€/km` (DESC).
    * Фильтр "Исключенные" (Blacklist).
    * Визуализация на карте: Маршрут A (пунктир) -> B (сплошная) -> C.
