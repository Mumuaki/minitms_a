# Use Case Specification: Authentication & Authorization

## 1. Введение
Модуль отвечает за безопасность доступа, управление сессиями и разграничение прав согласно матрице ролей (RBAC).

## 2. Use Cases

### UC-AUTH-01: Login (Вход в систему)
* **Actor:** Guest.
* **Input:** `email`, `password`.
* **Flow:**
    1. Проверка валидности email.
    2. Проверка блокировки аккаунта (если было > 5 неудачных попыток за 15 минут).
    3. Хеширование пароля (bcrypt) и сравнение с БД.
    4. Генерация JWT (Access Token) и Refresh Token.
    5. Запись события в `LoginLog`.
* **Output:** Токены, профиль пользователя, роль.

### UC-AUTH-02: Check Permission (Проверка прав)
* **Logic:** Middleware проверяет claims в JWT токене перед доступом к защищенным ресурсам.
* **Roles & Permissions:**
    * **Administrator:** Полный доступ (Управление пользователями, конфиг системы).
    * **Director:** Доступ к финансам, настройкам, но без создания пользователей.
    * **Dispatcher:** Поиск грузов, отправка email, создание заказов, редактирование Sheets. Нет доступа к финансам (только просмотр своих планов).
    * **Observer:** Только чтение (ReadOnly).

## 3. Требования безопасности
* **Session:** "Запомнить меня" — refresh token живет 30 дней.
* **Encryption:** Пароли никогда не передаются в открытом виде, используется TLS 1.3.
