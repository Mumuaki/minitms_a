# Спецификация: Trans.eu Scraping Service (spec_trans_eu.md)

## 1. Введение
Модуль предназначен для автоматического получения списка грузоперевозок с платформы `platform.trans.eu` путем эмуляции действий пользователя (Browser Automation / Scraping). Поскольку официальное API списка грузов недоступно, этот модуль является единственным источником данных о предложениях.

**Целевой URL**: `https://platform.trans.eu/exchange/offers`

---

## 2. Функциональные требования

### 2.1. Аутентификация
* Система должна использовать учетные данные (логин/пароль), сохраненные в настройках пользователя.
* Поддержка сохранения сессии (cookies) для минимизации количества входов.
* Автоматическая повторная авторизация при истечении сессии.

### 2.2. Управление фильтрами (Search Workflow)
Скрапер должен строго следовать процедуре установки фильтров, чтобы гарантировать корректную выдачу грузов.

#### Шаг 1: Инициализация полей локации (Loading / Unloading)
Процесс идентичен для полей "Загрузка" и "Разгрузка":
1.  **Клик**: Левой кнопкой мыши по полю `Loading` (или `Unloading`).
2.  **Ввод**: Ввод строки в формате: `{ISO-2 страны}, {Индекс} {Название города}`.
    * *Пример*: `CH, 8058 Zurich`.
3.  **Выбор из списка**: Ожидание появления динамического выпадающего списка.
    * **Правило выбора**: Выбрать элемент, содержащий **максимальное количество информации** (Страна + Индекс + Город + Район).
    * *Пример*: Из вариантов `CH, 8048` и `CH, 8048 Zurich, (Zurich) Zurich district` — выбрать второй.
4.  **Обработка Radius (Радиус поиска)**:
    * После выбора локации проверить наличие выпадающего списка радиуса (`+ 5 km` ... `+ 200 km`).
    * **Логика**:
        * Если локация точная (Страна + Индекс + Город) -> Установить **+ 75 km** (или значение из настроек фронтенда).
        * Если локация общая (Только страна) -> Список радиуса не появляется, действия не требуются.

#### Шаг 2: Настройка дат (Date Filters)
1.  Нажать кнопку `EXPAND FILTERS` (или `MORE FILTERS`) в правом верхнем углу.
2.  **Loading Date (Дата загрузки)**:
    * Левое поле (`From`): Установить **Текущую дату**.
    * Правое поле (`To`): Установить **Текущую дату**.
3.  **Unloading Date (Дата выгрузки)**:
    * Левое поле (`From`): Установить **Дата загрузки + 1 день** (или следующий рабочий день).
    * Правое поле (`To`): Установить **Дата загрузки + 1 день**.
4.  *Примечание*: Выбор даты осуществляется через UI-календарь.

#### Шаг 3: Параметры груза
1.  **Weight (Вес)**:
    * Поле `From`: Оставить пустым или `0`.
    * Поле `To`: Установить `0.9` (или согласно грузоподъемности конкретного ТС).

#### Шаг 4: Запуск поиска
1.  Нажать зеленую кнопку `Search` в правом верхнем углу.

### 2.3. Сбор данных (Extraction)
После загрузки результатов (DOM update) система должна извлечь данные из каждой карточки (Offer Card):

| Поле UI | Поле сущности `Cargo` | Преобразование |
|---|---|---|
| Место загрузки | `loading_place` | Извлечь Zip, City, Country |
| Место разгрузки | `unloading_place` | Извлечь Zip, City, Country |
| Дата публикации | `published_at` | Timestamp |
| Описание груза | `body_type`, `weight` | Парсинг строки (напр. "0.15 т, стандарт, штора") |
| Цена | `price` | Извлечь число, валюту привести к EUR |
| Расстояние (Trans) | `distance_trans_eu` | Извлечь число "km" |
| Компания | `company_name` | Название заказчика |
| Рейтинг | `company_rating` | Числовой рейтинг и кол-во отзывов |
| Trans ID | `external_id` | Уникальный ID предложения из атрибутов HTML |

### 2.4. Режимы работы
* **Интервальный**: Запуск каждые N минут (конфигурируется, min 5 мин).
* **По требованию**: Принудительное обновление по кнопке из UI.

## 3. Технические требования и ограничения

### 3.1. Нормализация данных
* **Валюта**: Все цены конвертировать в EUR. Если цена не указана, помечать как `null` или `0`.
* **Геокодирование**:
    * Полученные строковые адреса (`DE 80997 Мюнхен`) должны быть отправлены в Nominatim API (OpenStreetMap) для получения координат (`lat`, `lon`).
    * Это необходимо для корректного расчета маршрутов и отображения на карте MiniTMS.

### 3.2. Обработка ошибок
* **Изменение верстки**: При изменении CSS-селекторов на Trans.eu система должна логировать критическую ошибку и уведомлять администратора.
* **Капча/Блокировка**: Реализовать эмуляцию задержек (human-like behavior) между кликами (random 500ms - 1500ms).

### 3.3. Производительность
* Время полного цикла (Логин -> Фильтр -> Парсинг 50 грузов) не должно превышать 30-45 секунд.
