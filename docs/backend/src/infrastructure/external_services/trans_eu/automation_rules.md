# Правила автоматизации ввода данных для Trans.eu

## 1. Ввод местоположения (Loading/Unloading Place)

### 1.1. Алгоритм и форматы ввода
Ввод текста в поля "Загрузка" и "Разгрузка" должен производиться строго в указанном ниже порядке с соблюдением знаков пунктуации. Каждая попытка ввода должна сопровождаться ожиданием появления модального окна.

**Основной приоритет (на английском языке):**
1. `{ISO-2 страны}, {Индекс}, {Город}` (напр. `PL, 00-001, Warszawa`)
2. `{ISO-2 страны}, {Город}` (напр. `PL, Warszawa`)
3. `{Город}` (напр. `Warszawa`)

**Обязательное условие:** Результатом правильного ввода должно быть появление модального окна с перечнем активных полей для подтверждения и фиксации корректного ввода.

**Резервный порядок (если модальное окно не появляется):**
1. `{Город}` на русском языке (напр. `Варшава`)
2. `{ISO-2 страны}` на латинице, `{Город}` на русском языке (напр. `PL, Варшава`)

### 1.2. Выбор из списка
- Ожидание появления динамического выпадающего списка (dropdown).
- Выбор элемента, содержащего максимально полную информацию (Страна + Индекс + Город).

## 2. Работа с датами (Calendar Handling)

### 2.1. Алгоритм выбора даты
При установке дат загрузки и выгрузки использование прямой печати (typing) в поле ЗАПРЕЩЕНО. Необходимо следовать сценарию:
1. Клик в поле ввода для вызова календаря.
2. **Проверка текущего месяца**: Считать заголовок. Если месяц/год не совпадают с целевым, использовать стрелки навигации (`<` / `>`) для перехода.
3. Поиск элемента соответствующего дня (через `data-ctx-id` или точное текстовое совпадение во избежание выбора дня из соседнего месяца).
4. **Клик левой кнопкой мыши** на выбранной дате для фиксации значения.

### 2.2. Формирование диапазонов
Для ввода диапазона дат необходимо последовательно вызвать календарь для каждой границы диапазона (ОТ и ДО), выполняя алгоритм из п. 2.1.

### 2.3. Правила валидации (Business Rules) -- Строго соблюдать!
1. **Хронология диапазона**: Для любой пары дат, дата `From` (От) не может быть позже даты `To` (До).
2. **Логическая согласованность**: Любая дата разгрузки (`Unloading Date`, будь то `From` или `To`) **не может быть раньше** любой даты загрузки (`Loading Date`). Попытка задать разгрузку раньше загрузки должна блокироваться на уровне валидации перед взаимодействием с UI.
