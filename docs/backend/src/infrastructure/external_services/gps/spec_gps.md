# Спецификация: GPS Integration Service (spec_gps.md)

## 1. Введение
Модуль отвечает за получение фактических данных о местоположении и пробеге транспортных средств от внешних телематических платформ. Эти данные являются критически важными для двух бизнес-процессов:
1.  **Поиск грузов**: Определение точки старта (Point A) для фильтрации предложений по радиусу.
2.  **Финансовое планирование**: Расчет фактического пробега за день/месяц для сравнения с планом.

---

## 2. Поддерживаемые Платформы
Система должна поддерживать интеграцию через API со следующими провайдерами:
* **Wialon** (Hosting / Local)
* **Navixy**
* **GPS-Trace**

---

## 3. Сценарии Использования (Workflows)

### 3.1. Обновление местоположения (On-Demand Location Update)
Этот процесс инициируется пользователем через UI (кнопка "Обновить местоположение") для получения актуальной точки старта поиска.

1.  **Запрос**: Frontend отправляет запрос `POST /api/v1/fleet/{id}/sync-location`.
2.  **API Call**: Backend запрашивает у GPS-провайдера текущие координаты (`lat`, `lon`) и время фиксации.
3.  **Reverse Geocoding**: Полученные координаты отправляются в Google Maps Geocoding API для получения человекочитаемого адреса (Населенный пункт, Страна).
4.  **Сохранение**: Данные записываются в сущность `VehiclePosition` с обновлением ссылки в `Vehicle`.
5.  **Ответ**: Возврат обновленных данных на Frontend для отображения в хедере поиска.

### 3.2. Сбор данных о пробеге (Daily Mileage Sync)
Этот процесс работает в фоновом режиме (Scheduled Task) для финансового модуля.

1.  **Расписание**: Ежедневно в 00:05 (для закрытия предыдущего дня).
2.  **Логика**:
    * Получить показания одометра на `00:00:00` начала дня.
    * Получить показания одометра на `23:59:59` конца дня.
    * `Фактический пробег` = `Odometer(End) - Odometer(Start)`.
3.  **Сохранение**: Запись значения в таблицу статистики выполнения плана (`PlanExecution`).

---

## 4. Модель Данных

### 4.1. Входные данные (От провайдера)
* `tracker_id`: Уникальный ID устройства.
* `pos`: `{ x: float, y: float }` (Lat/Lon).
* `t`: Timestamp (Unix time).
* `speed`: Integer (km/h) - опционально для статуса "В движении".
* `mileage`: BigInt (Total Odometer meteres).

### 4.2. Хранение (VehiclePosition)
Согласно:
* `vehicle_id`: UUID.
* `latitude`: Decimal(10, 8).
* `longitude`: Decimal(11, 8).
* `place_name`: String (напр. "Bratislava").
* `country`: String (ISO-2, напр. "SK").
* `measured_at`: Timestamp (Время GPS).
* `updated_at`: Timestamp (Время сервера).

---

## 5. Обработка ошибок
* **API провайдера недоступен**: Повторить запрос 3 раза с интервалом 2 секунды. Если неудача — вернуть пользователю ошибку "GPS недоступен, используются последние известные координаты".
* **Устаревшие данные**: Если время `measured_at` отличается от текущего более чем на 4 часа — выдать предупреждение в UI "Данные GPS устарели".
* **Точность геокодирования**: В приграничных зонах использовать код страны, возвращаемый Geocoder, как приоритетный для фильтрации.
