# Политика и спецификация безопасности (Security)

**Версия:** 1.0

## 1. Аутентификация и Управление сессиями

### 1.1 Механизм входа
* Используется **JWT (Access Token + Refresh Token)**.
* **Access Token:**
    * Срок жизни: 15 минут.
    * Подпись: RS256 (Asymmetric keys).
    * Payload: `sub` (userId), `role`, `iat`, `exp`.
* **Refresh Token:**
    * Срок жизни: 30 дней (при флаге "Запомнить меня").
    * Хранение: HttpOnly Cookie для защиты от XSS.
    * Ротация: При использовании Refresh токен обновляется, старый аннулируется.

**Поведение при блокировке:**
* Блокировка аккаунта на **15 минут** после **5 неудачных попыток** входа.
* Инкремент счётчика при каждой неудачной попытке.
* Сброс счётчика при успешном входе.
* Rate Limiting на эндпоинт `/auth/login`: 10 запросов в минуту с одного IP.

### 2.2 RBAC (Role-Based Access Control)
Реализован Role-Based Access Control. Матрица доступа строго соответствует спецификации.

### Роли и области видимости
1.  **Administrator**: Полный доступ (Управление пользователями, ролями, настройками системы).
2.  **Director**: Управленческий доступ (Установка планов, просмотр финансов, настройки интеграций). Нет доступа к администрированию пользователей.
3.  **Dispatcher**: Операционный доступ (Поиск грузов, Fleet, Email, Google Sheets - редактирование). Нет доступа к финансовым планам (только просмотр своих показателей, если применимо) и администрированию.
4.  **Observer**: Только чтение (Сводки, отчеты). Нет доступа к редактированию данных и отправке писем.

## 3. Шифрование и хеширование

### 3.1 Хеширование
* **Пароли пользователей:** Хеширование с использованием `bcrypt` (salt rounds >= 10).
* **Токены доступа к API:** Хранятся хешированными (SHA-256 при необходимости).

### 3.2 Шифрование данных в покое
* **Credentials (Trans.eu, GPS, Google):** Хранятся в зашифрованном виде в БД (AES-256, ключ в переменных окружения).
* **Sensitive Fields:** Например, контакты клиентов могут быть зашифрованы при необходимости GDPR.

### 3.3 GDPR Compliance
* Персональные данные водителей и клиентов могут быть анонимизированы по запросу.
* Логирование действий пользователей (Audit Log) хранится 1 год.

## 4. Интеграции и доступ к внешним API

### 4.1 Безопасность парсинга (Trans.eu)
* **Имитация человека:** Использование `User-Agent` реальных браузеров, рандомизация задержек между запросами.
* **Cookie Management:** Безопасное хранение сессионных куки Trans.eu, автоматическое обновление без передачи пароля в открытом виде при каждом запросе.
* **Proxy:** Поддержка ротации прокси-серверов для избежания блокировки IP (опционально).
### 4.2 Email Security
* **Rate Limiting:** Жесткое ограничение на уровне приложения — **максимум 50 писем в час** на пользователя.
* **Throttling:** Минимальный интервал между отправками — **30 секунд**.
* Блокировка массовой рассылки при попытке превышения лимитов.

### 4.3 Google Integration
* Использование **OAuth 2.0**. Приложение не хранит пароли от Google аккаунтов.
* Минимально необходимые Scope:
    * `https://www.googleapis.com/auth/spreadsheets`
    * `https://www.googleapis.com/auth/drive.file`
* Токены доступа хранятся в защищенном виде.
