# Политика и спецификация безопасности (Security)

**Версия:** 1.0

## 1. Аутентификация и Управление сессиями

### 1.1 Механизм входа
* Используется **JWT (Access Token + Refresh Token)**.
* **Access Token:**
    * Срок жизни: 15 минут.
    * Подпись: RS256 (Asymmetric keys).
    * Payload: `sub` (userId), `role`, `iat`, `exp`.
* **Refresh Token:**
    * [cite_start]Срок жизни: 30 дней (при флаге "Запомнить меня")[cite: 5].
    * Хранение: HttpOnly Secure Cookie (для Web) или Secure Storage (для Mobile).
    * Ротация: При каждом использовании выдается новый Refresh Token (Refresh Token Rotation).

### 1.2 Защита от Brute-force
* [cite_start]Блокировка аккаунта на **15 минут** после **5 неудачных попыток** входа[cite: 5].
* Rate Limiting на эндпоинт `/auth/login`: 10 запросов в минуту с одного IP.

## 2. Авторизация (RBAC)
Реализован Role-Based Access Control. [cite_start]Матрица доступа строго соответствует спецификации[cite: 5].

### Роли и области видимости
1.  **Administrator**: Полный доступ (`*`). Управление пользователями и системными конфигами.
2.  **Director**: Полный доступ к бизнес-функциям, финансам, настройкам Trans.eu. Нет доступа к созданию пользователей.
3.  **Dispatcher**: Операционный доступ (Поиск грузов, Fleet, Email, Google Sheets - редактирование). Нет доступа к финансовым планам (только просмотр своих показателей, если применимо) и администрированию.
4.  **Observer**: Только чтение (Read-only). Дашборды, статусы ТС.

## 3. Защита данных (Data Protection)

### 3.1 Шифрование
* [cite_start]**Пароли пользователей:** Хеширование с использованием `bcrypt` (salt rounds >= 10)[cite: 5].
* **Учетные данные Trans.eu:** Хранятся в БД в зашифрованном виде (AES-256-GCM). Ключ шифрования хранится в переменных окружения (Vault/Secret Manager), но не в коде.
* **Передача данных:** Весь трафик шифруется TLS 1.3.

### 3.2 GDPR Compliance
* Персональные данные водителей и клиентов могут быть анонимизированы по запросу.
* Логирование действий пользователей (Audit Log) хранится 1 год.

## 4. Безопасность интеграций

### 4.1 Безопасность парсинга (Trans.eu)
* **Имитация человека:** Использование `User-Agent` реальных браузеров, рандомизация задержек между запросами.
* **Cookie Management:** Безопасное хранение сессионных куки Trans.eu, автоматическое обновление без передачи пароля в открытом виде при каждом запросе.
* **Proxy:** Поддержка ротации прокси-серверов для избежания блокировки IP (опционально).

### 4.2 Email Security
* [cite_start]**Rate Limiting:** Жесткое ограничение на уровне приложения — **максимум 50 писем в час** на пользователя[cite: 5].
* [cite_start]**Throttling:** Минимальный интервал между отправками — **30 секунд**[cite: 5].
* Блокировка массовой рассылки при попытке превышения лимитов.

### 4.3 Google Integration
* Использование **OAuth 2.0**. Приложение не хранит пароли от Google аккаунтов.
* Минимально необходимые Scope:
    * `https://www.googleapis.com/auth/spreadsheets`
    * `https://www.googleapis.com/auth/drive.file`
* Токены доступа хранятся в защищенном виде.
