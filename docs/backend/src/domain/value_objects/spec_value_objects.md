# Value Objects Specification

Спецификация объектов-значений (Immutable Value Objects). Эти объекты инкапсулируют атрибуты и валидацию, гарантируя целостность данных во всем приложении.

## 1. Базовые типы

### `Money`
Представляет денежную сумму.
* **Поля:** `amount` (BigDecimal), `currency` (Enum: EUR).
* **Инварианты:**
    * `amount` не может быть отрицательным (для цен).
    * Все расчеты внутри системы ведутся в EUR (BR-001).
* **Методы:** `add(Money)`, `subtract(Money)`, `multiply(factor)`, `perKm(distanceInKm)`.

### `GeoLocation`
Географические координаты.
* **Поля:** `latitude` (float), `longitude` (float).
* **Инварианты:**
    * Lat: -90 до 90.
    * Lon: -180 до 180.
* **Методы:**
    * `distanceTo(other: GeoLocation): Distance` — расчет по формуле Haversine или через OSM Adapter.

### `Distance`
Физическое расстояние.
* **Поля:** `value` (float), `unit` (Enum: KM).
* **Инварианты:** Значение ≥ 0. Основная единица — км (BR-002).

## 2. Параметры груза и транспорта

### `Dimensions`
Размеры (кузова или груза).
* **Поля:** `length` (float), `width` (float), `height` (float).
* **Инварианты:** Все значения > 0. Хранение в метрах (BR-003).
* **Методы:**
    * `volume(): float` (м³)
    * `fitsIn(other: Dimensions): boolean` — проверка, помещается ли груз в ТС.

### `Weight`
Вес.
* **Поля:** `value` (float), `unit` (Enum: KG).
* **Инварианты:** Хранение в кг (BR-004). В UI может конвертироваться в тонны.

### `BodyType`
Тип кузова.
* **Поля:** `code` (Enum).
* **Значения:** `TILT` (Тент), `FRIDGE` (Рефрижератор), `BOX` (Фургон), `MEGA` и др.
* **Валидация:** Должен соответствовать справочнику Trans.eu.

## 3. Расчетные показатели

### `ProfitabilityRate`
Ставка за километр.
* **Поля:** `value` (float).
* **Логика:**
    * Вычисляется как: `CargoPrice / (DeadheadDistance + LoadedDistance)`.
* **Методы:**
    * `getStatus(): ProfitabilityStatus` — Возвращает статус цвета согласно спецификации:
        * 🔴 `BAD` (< 0.54)
        * ⚪ `LOW` (0.54 - 0.59)
        * 🟡 `MEDIUM` (0.60 - 0.79)
        * 🟢 `GOOD` (≥ 0.80)

## 4. Контакты и идентификаторы

### `EmailAddress`
* **Поля:** `value` (string).
* **Инварианты:** Валидация регулярным выражением (формат email).

### `LicensePlate`
Госномер автомобиля.
* **Поля:** `value` (string).
* **Методы:** Нормализация (удаление пробелов, перевод в верхний регистр).

### `TransEuId`
Уникальный идентификатор предложения в системе Trans.eu.
* **Поля:** `value` (string).
* **Пример:** Идентификатор из URL или API ответа.
